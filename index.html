Query Data with DuckDB WASM




Instructions:
  • Select whether you want to query a Parquet file from a URL or a local file.
  • If using a URL, paste the URL in the text box. The server hosting the file must have CORS enabled.
  • If using a local file, select the file using the file picker.
  • Write your SQL query in the custom query text area. The table name is my_data.parquet.
  • Click "Run Query" to see the results.
Important Note on CORS: For DuckDB WASM to access a Parquet file from a URL, the server hosting the file (e.g., AWS S3, Azure Blob Storage, Google Cloud Storage) must be configured with Cross-Origin Resource Sharing (CORS) headers. These headers need to allow requests from the domain where this page is hosted, or be set to allow all origins (e.g., Access-Control-Allow-Origin: *). Without proper CORS configuration, the browser will block the request.
